YouTube Top-K Architecture Component 3: trust boundary for ranking signals
Component 3

Validation, Deduplication, Fraud Filter

Component 3 converts untrusted raw playback activity into trusted ranking signals. It decides whether an event can affect hot top-k rankings, whether it should be rejected, or whether it should be quarantined for later review.

InputRawWatchEvent from the event gateway.
DecisionAccept, reject, quarantine, or route to backfill.
OutputValidViewEvent into Component 4.
DefaultIf trust is uncertain, do not affect hot rankings.

1. High-Level System Interactions

Component 3 sits between the gateway and the durable event stream. It uses short-lived state stores and metadata services, then emits separate streams for valid, rejected, suspicious, and late events.

3. Java 21 Implementation Sketch

This is the production-shaped code boundary for Component 3. The first implementation can keep the stores in-memory for tests, but the interfaces map directly to Redis/Valkey, session state, policy metadata, and Kafka producers.

Event models and decisions

package com.systemdesign.youtube.topk.validation;

import java.time.Instant;
import java.util.Map;

public record RawWatchEvent(
    String eventId,
    String videoId,
    String sessionId,
    String userId,
    Instant eventTime,
    Instant ingestTime,
    long watchMs,
    String country,
    String device,
    Map<String, String> attributes
) {}

public record ValidViewEvent(
    String eventId,
    String videoId,
    Instant eventTime,
    String country,
    long validViews,
    long watchTimeMs,
    String validationVersion,
    String fraudDecision
) {}

public enum DecisionType {
  ACCEPT,
  REJECT,
  QUARANTINE,
  BACKFILL
}

public record ValidationDecision(
    DecisionType type,
    String reason,
    ValidViewEvent validEvent
) {
  static ValidationDecision accept(ValidViewEvent event) {
    return new ValidationDecision(DecisionType.ACCEPT, "accepted", event);
  }

  static ValidationDecision reject(String reason) {
    return new ValidationDecision(DecisionType.REJECT, reason, null);
  }

  static ValidationDecision quarantine(String reason) {
    return new ValidationDecision(DecisionType.QUARANTINE, reason, null);
  }

  static ValidationDecision backfill(String reason) {
    return new ValidationDecision(DecisionType.BACKFILL, reason, null);
  }
}

public record VideoPolicy(boolean rankable, String reason) {}

4. Component 3 Processor Code

The processor is intentionally boring: validate in order, fail closed for hot rankings, and publish exactly one outcome stream per raw event. Heavy fraud review and correction happen asynchronously. 

package com.systemdesign.youtube.topk.validation;

import java.time.Duration;
import java.time.Instant;
import java.util.Set;

public final class Component3Validator {
  private static final String VALIDATION_VERSION = "valid-view-v1";
  private static final Duration EVENT_DEDUPE_TTL = Duration.ofMinutes(30);
  private static final Duration MAX_FUTURE_SKEW = Duration.ofSeconds(30);
  private static final Duration HOT_PATH_ALLOWED_LATENESS = Duration.ofMinutes(10);
  private static final Duration BACKFILL_ALLOWED_LATENESS = Duration.ofHours(24);
  private static final long MIN_VALID_WATCH_MS = 30_000;
  private static final Set<String> ALLOWED_DEVICES = Set.of("web", "mobile", "tv", "embed");

  private final DedupeStore dedupeStore;
  private final SessionTrustStore sessionTrustStore;
  private final FraudFilter fraudFilter;
  private final PolicyStore policyStore;
  private final ValidationPublisher publisher;

  public Component3Validator(
      DedupeStore dedupeStore,
      SessionTrustStore sessionTrustStore,
      FraudFilter fraudFilter,
      PolicyStore policyStore,
      ValidationPublisher publisher
  ) {
    this.dedupeStore = dedupeStore;
    this.sessionTrustStore = sessionTrustStore;
    this.fraudFilter = fraudFilter;
    this.policyStore = policyStore;
    this.publisher = publisher;
  }

  public ValidationDecision validateAndPublish(RawWatchEvent event, Instant now) {
    ValidationDecision decision = validate(event, now);

    switch (decision.type()) {
      case ACCEPT -> publisher.publishValid(decision.validEvent());
      case REJECT -> publisher.publishRejected(event, decision.reason());
      case QUARANTINE -> publisher.publishSuspicious(event, decision.reason());
      case BACKFILL -> publisher.publishLate(event, decision.reason());
    }

    return decision;
  }

  public ValidationDecision validate(RawWatchEvent event, Instant now) {
    ValidationDecision schema = validateSchema(event);
    if (schema.type() != DecisionType.ACCEPT) return schema;

    ValidationDecision time = validateEventTime(event, now);
    if (time.type() != DecisionType.ACCEPT) return time;

    SessionTrust trust = sessionTrustStore.lookup(event.sessionId(), event.userId());
    if (!trust.trusted()) {
      return ValidationDecision.quarantine("untrusted_session:" + trust.reason());
    }

    String dedupeKey = "dedupe:event:" + event.eventId();
    boolean firstSeen = dedupeStore.putIfAbsent(dedupeKey, EVENT_DEDUPE_TTL);
    if (!firstSeen) {
      return ValidationDecision.reject("duplicate_event");
    }

    if (event.watchMs() < MIN_VALID_WATCH_MS) {
      return ValidationDecision.reject("watch_too_short");
    }

    FraudDecision fraud = fraudFilter.evaluate(event);
    if (!fraud.allowed()) {
      return ValidationDecision.quarantine("fraud_risk:" + fraud.reason());
    }

    VideoPolicy policy = policyStore.lookupVideoPolicy(event.videoId());
    if (!policy.rankable()) {
      return ValidationDecision.reject("not_rankable:" + policy.reason());
    }

    ValidViewEvent valid = new ValidViewEvent(
        event.eventId(),
        event.videoId(),
        event.eventTime(),
        event.country(),
        1L,
        event.watchMs(),
        VALIDATION_VERSION,
        fraud.reason()
    );

    return ValidationDecision.accept(valid);
  }

  private ValidationDecision validateSchema(RawWatchEvent event) {
    if (event == null) return ValidationDecision.reject("null_event");
    if (isBlank(event.eventId())) return ValidationDecision.reject("missing_event_id");
    if (isBlank(event.videoId())) return ValidationDecision.reject("missing_video_id");
    if (isBlank(event.sessionId())) return ValidationDecision.reject("missing_session_id");
    if (event.eventTime() == null) return ValidationDecision.reject("missing_event_time");
    if (event.ingestTime() == null) return ValidationDecision.reject("missing_ingest_time");
    if (event.watchMs() < 0) return ValidationDecision.reject("negative_watch_ms");
    if (isBlank(event.country())) return ValidationDecision.reject("missing_country");
    if (!ALLOWED_DEVICES.contains(event.device())) return ValidationDecision.reject("invalid_device");
    return ValidationDecision.accept(null);
  }

  private ValidationDecision validateEventTime(RawWatchEvent event, Instant now) {
    if (event.eventTime().isAfter(now.plus(MAX_FUTURE_SKEW))) {
      return ValidationDecision.reject("event_time_too_far_in_future");
    }

    Duration lateness = Duration.between(event.eventTime(), now);
    if (lateness.compareTo(BACKFILL_ALLOWED_LATENESS) > 0) {
      return ValidationDecision.reject("event_too_old");
    }

    if (lateness.compareTo(HOT_PATH_ALLOWED_LATENESS) > 0) {
      return ValidationDecision.backfill("late_for_hot_ranking");
    }

    return ValidationDecision.accept(null);
  }

  private static boolean isBlank(String value) {
    return value == null || value.isBlank();
  }
}

5. State And Hot-Node Strategy

Event-id dedupe distributes well because event IDs are random. Hot-node risk comes from low-cardinality or viral keys such as IP, video ID, country, and creator. Component 3 avoids direct per-video counter writes and salts hot abuse counters.

7. Failure Behavior

The failure policy is conservative: if we cannot trust the event, it should not affect hot rankings. Some failures degrade to cached/simple rules; others quarantine or reject.

Node Failure Hot Ranking Behavior Follow-Up
Schema validatorBad schema or parser bugReject malformed events; canary rollback on rate shiftDead-letter queue and schema version audit
Auth/session validatorSession store unavailableUse short local cache; otherwise quarantineReplay quarantined events if trust can be proven
DeduplicatorRedis shard hot or unavailableUse local dedupe fallback; uncertain events quarantineScale shards, shorten TTL, or enable probabilistic filter
Fraud filterRules/model unavailableFall back to simple rules; quarantine high-risk trafficAsync review and correction snapshots
